# Product Manager Skill - OpenCode 使用示例

## 场景：手机号登录功能 PRD 编写

### 前置准备

```bash
# 1. 进入项目目录
cd /path/to/project

# 2. 启动 OpenCode
opencode
```

### 步骤 1：创建 PRD 框架

在 OpenCode 对话中：

```
skill(name: product-manager)
```

OpenCode 会加载 Product Manager Skill，然后输入：

```
请帮我创建一个手机号登录功能的 PRD 框架。

## 背景

用户反馈当前登录流程太复杂，仅支持账号密码登录，忘记密码后需要邮件申诉，流程过长。

## 业务目标

- 提升登录转化率 15%
- 降低客服咨询量 30%

## 团队信息

- 产品经理：@zhangsan
- 优先级：P0
- 期望上线日期：2024-01-30
```

### 步骤 2：完善 PRD

AI 生成 PRD 框架后，继续完善：

```
请完善以下 PRD 文档：

1. 完善用户故事表格（至少 5 个故事，覆盖所有角色）
2. 为每个功能需求添加验收条件（可量化、可测试）
3. 补充数据埋点设计
4. 完善非功能需求（性能、安全、监控）

## 参考信息

- 目标用户：新用户（未设置过密码）、老用户（忘记密码）
- 登录方式：手机号 + 6 位短信验证码
- 验证码有效期：5 分钟
- 支持"7 天内自动登录"选项
```

### 步骤 3：输出 PRD 文档

AI 生成的完整 PRD：

```markdown
---
id: PRD-2024-001
title: 手机号登录功能
product-manager: @zhangsan
create-date: 2024-01-15
priority: P0
status: draft
---

# 需求背景

## 问题描述

当前登录流程存在以下问题：
1. 仅支持账号密码登录，用户需要记忆密码
2. 忘记密码流程复杂，需要邮件申诉（平均耗时 2 天）
3. 新用户首次使用需要设置密码，流程过长，流失率 35%

## 目标用户

| 用户类型 | 描述 | 占比 |
|----------|------|------|
| 新用户 | 未设置过密码，希望快速体验产品 | 40% |
| 老用户 | 忘记密码，无法快速登录 | 25% |
| 所有用户 | 希望简化登录流程 | 100% |

## 业务目标

| 指标 | 当前值 | 目标值 | 提升 |
|------|--------|--------|------|
| 登录转化率 | 65% | 80% | +15% |
| 客服咨询量 | 100 次/天 | 70 次/天 | -30% |
| NPS | 35 | 45 | +10 分 |

# 用户故事

| ID | 作为 | 我想要 | 以便于 | 优先级 |
|----|------|--------|--------|--------|
| US-001 | 用户 | 通过手机号 + 验证码登录 | 无需记忆密码快速登录 | P0 |
| US-002 | 用户 | 获取短信验证码 | 完成登录验证 | P0 |
| US-003 | 用户 | 60 秒后重新发送验证码 | 防止验证码未到达 | P1 |
| US-004 | 用户 | 验证码错误时看到明确提示 | 知道如何修正 | P1 |
| US-005 | 用户 | 登录成功后跳转到首页 | 继续使用系统 | P1 |
| US-006 | 用户 | 选择 7 天内自动登录 | 避免重复登录 | P2 |

# 功能需求

## FR-001: 手机号登录

### 描述

用户通过手机号 + 6 位数字验证码完成登录。

### 验收条件

- [ ] 支持 11 位中国大陆手机号（13x/14x/15x/16x/17x/18x/19x）
- [ ] 验证码为 6 位数字
- [ ] 验证码有效期 5 分钟
- [ ] 同一手机号 60 秒内只能获取一次验证码
- [ ] 验证码错误时提示"验证码错误，请重新输入"
- [ ] 验证码过期时提示"验证码已过期，请重新获取"
- [ ] 登录成功后跳转回登录前页面或首页

## FR-002: 验证码获取

### 描述

用户点击"获取验证码"按钮后，系统发送短信验证码到用户手机。

### 验收条件

- [ ] 点击按钮前校验手机号格式
- [ ] 手机号格式错误时提示"请输入正确的手机号"
- [ ] 发送成功后按钮进入 60 秒倒计时
- [ ] 倒计时结束后显示"重新发送"
- [ ] 发送失败时显示错误原因

## FR-003: 登录状态保持

### 描述

用户登录成功后，可选择"7 天内自动登录"。

### 验收条件

- [ ] 默认不勾选自动登录
- [ ] 勾选后 7 天内无需重复登录
- [ ] 敏感操作（如修改密码）需重新验证
- [ ] 用户可手动退出登录

# 非功能需求

## 性能要求

| 指标 | 目标值 | 测量方式 |
|------|--------|----------|
| 登录接口 P95 延迟 | < 500ms | 监控系统 |
| 验证码发送延迟 | < 5s | 短信服务商 |
| 支持 QPS | > 1000 | 压力测试 |

## 安全要求

- [ ] 验证码尝试次数限制：5 次/小时
- [ ] 同一手机号发送频率限制：10 次/天
- [ ] 防刷机制：IP 维度限流
- [ ] 短信内容不含敏感信息
- [ ] 登录日志记录（IP、User-Agent、时间）

## 监控要求

- [ ] 登录成功率监控（告警阈值：< 95%）
- [ ] 验证码发送成功率监控（告警阈值：< 90%）
- [ ] 异常登录行为告警（如短时间内多次失败）

# 数据埋点

| 事件 | 触发时机 | 参数 | 用途 |
|------|----------|------|------|
| login_page_view | 打开登录页面 | source（来源页面） | 分析登录页访问量 |
| send_code_click | 点击发送验证码 | phone（脱敏） | 分析验证码请求量 |
| send_code_success | 验证码发送成功 | phone（脱敏） | 分析短信服务成功率 |
| send_code_fail | 验证码发送失败 | phone（脱敏）, reason | 分析失败原因 |
| login_submit | 提交登录 | phone（脱敏） | 分析登录尝试量 |
| login_success | 登录成功 | phone（脱敏）, duration | 分析登录转化率 |
| login_fail | 登录失败 | phone（脱敏）, reason | 分析失败原因 |

# 附录

```
