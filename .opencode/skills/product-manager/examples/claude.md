# Product Manager Skill - Claude 使用示例

## 场景：手机号登录功能 PRD 编写

### 前置准备

```bash
# 1. 配置 Claude Desktop（首次使用）
# 在 ~/claude_desktop_config.md 中添加：

# Product Manager Skill 定义
---
## 角色：产品经理

你是一名资深产品经理，拥有 10 年互联网产品经验。

## 核心能力

1. 需求分析与拆解
2. 用户故事编写（作为...我想要...以便于...）
3. 产品文档撰写（结构化 PRD）
4. 竞品分析
5. 数据埋点设计

## 输出规范

- 使用 Markdown 格式
- 包含 YAML frontmatter（id, title, product-manager, create-date, priority, status）
- 用户故事用表格呈现
- 验收条件用复选框列表
- 业务目标尽量量化
---
```

### 步骤 1：创建 PRD 框架

在 Claude 对话中：

```
我是一名产品经理，需要创建手机号登录功能的 PRD。

## 原始需求

用户反馈当前登录流程太复杂：
- 仅支持账号密码登录，用户需要记忆密码
- 忘记密码后需要邮件申诉，流程过长（平均 2 天）
- 新用户首次使用需要设置密码，流失率 35%

## 业务目标

- 提升登录转化率 15%（从 65% 到 80%）
- 降低客服咨询量 30%（从 100 次/天到 70 次/天）
- 提升 NPS 10 分（从 35 到 45）

## 产品信息

- 产品名称：XXX 平台
- 目标用户：新用户（40%）、忘记密码的老用户（25%）
- 登录方式：手机号 + 6 位短信验证码
- 优先级：P0
- 期望上线：2024-01-30

请帮我创建结构化的 PRD 文档。
```

### 步骤 2：完善 PRD

Claude 生成 PRD 框架后，继续完善：

```
很好！请继续完善以下部分：

1. **用户故事** - 至少 5 个故事，覆盖所有角色
   - 使用表格格式：ID | 作为 | 我想要 | 以便于 | 优先级
   
2. **功能需求** - 为每个功能添加验收条件
   - FR-001: 手机号登录
   - FR-002: 验证码获取
   - FR-003: 登录状态保持

3. **非功能需求**
   - 性能要求（P95 延迟、QPS）
   - 安全要求（限流、日志）
   - 监控要求（告警阈值）

4. **数据埋点** - 完整的事件埋点设计
   - 事件名、触发时机、参数、用途
```

### 步骤 3：竞品分析

```
请进行竞品分析，对比以下产品的登录功能：

1. **微信** - 手机号 + 验证码登录
2. **支付宝** - 手机号 + 验证码登录
3. **淘宝** - 手机号 + 验证码登录

## 分析维度

- 功能完整性
- 用户体验
- 技术实现
- 可借鉴的设计

## 输出格式

Markdown 对比表格 + 差异化建议
```

### 步骤 4：验收测试用例

```
请根据 PRD 编写验收测试用例。

## 要求

- 每个功能点至少 3 个测试用例
- 包含正常流程和异常流程
- 用 Given-When-Then 格式描述
- 标注优先级（P0/P1/P2）

## 输出格式

| 用例 ID | 需求 ID | 类型 | 优先级 | 前置条件 | 步骤 | 预期结果 |
```

### 步骤 5：保存文档

将 AI 生成的 PRD 保存到项目：

```bash
# 创建目录
mkdir -p docs/prd

# 保存 PRD 文档
cat > docs/prd/mobile-login.md << 'PRD_EOF'
{粘贴 AI 生成的 PRD 内容}
PRD_EOF

# 提交 Git
git add docs/prd/mobile-login.md
git commit -m "feat: 添加手机号登录功能 PRD"
```

## 完整对话示例

### 用户输入

```
请帮我创建手机号登录功能的 PRD。

## 背景

当前登录流程存在问题：
- 仅支持账号密码登录
- 忘记密码流程复杂（邮件申诉，2 天）
- 新用户流失率 35%

## 目标

- 登录转化率：65% → 80%（+15%）
- 客服咨询：100 → 70 次/天（-30%）
- NPS: 35 → 45（+10 分）

## 方案

- 手机号 + 6 位短信验证码登录
- 验证码有效期 5 分钟
- 支持 7 天内自动登录

请输出完整的 PRD 文档，包含：
1. 需求背景（问题、目标用户、业务目标）
2. 用户故事（表格格式）
3. 功能需求（含验收条件）
4. 非功能需求（性能、安全、监控）
5. 数据埋点
```

### Claude 输出

Claude 会生成完整的 PRD 文档（约 1000 行），包含：

```markdown
---
id: PRD-2024-001
title: 手机号登录功能
product-manager: @zhangsan
create-date: 2024-01-15
priority: P0
status: draft
---

# 需求背景

## 问题描述

当前登录流程存在以下问题：
...

# 用户故事

| ID | 作为 | 我想要 | 以便于 | 优先级 |
|----|------|--------|--------|--------|
| US-001 | 用户 | 通过手机号 + 验证码登录 | 无需记忆密码 | P0 |
...
```

## 质量检查

使用检查清单验证 PRD：

```
请检查以上 PRD 是否满足以下要求：

- [ ] 需求背景清晰，有数据支撑（如流失率 35%）
- [ ] 用户故事完整，覆盖所有角色（新用户、老用户）
- [ ] 验收条件可量化、可测试（如"支持 11 位手机号"）
- [ ] 优先级明确，MVP 范围清晰（P0/P1/P2）
- [ ] 非功能需求完整（性能、安全、监控）
- [ ] 避免使用"可能"、"大概"等模糊词汇
- [ ] 数据埋点完整（7 个事件）

请逐项检查并指出不足之处。
```
