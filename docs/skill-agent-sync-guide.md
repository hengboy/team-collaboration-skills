# Skill 与 Agent 同步指南

## 单一事实源

- `skills/{name}/SKILL.md` 是事实源
- `agents/{name}/AGENT.md` 是派生物

派生规则：

- 允许精简示例、扩展说明、长模板
- 不允许改写核心能力契约
- 当前以 `## 核心契约（供 AGENT 派生）` 作为同步锚点

## 必须保持一致的内容

- 角色定位
- 必须输入
- 可选输入
- 输出文件
- 执行规则
- 质量检查
- 下一步流程

## 推荐结构

每个 `SKILL.md` 与 `AGENT.md` 都应包含：

- `## 角色定位`
- `## 适用场景`
- `## 输入要求`
- `### 必须输入`
- `### 可选输入`
- `## 输出规范`
- `### 输出文件`
- `## 执行规则`
- `## 质量检查`
- `## 下一步流程`
- `## 核心契约（供 AGENT 派生）`

## 同步流程

1. 先修改 `SKILL.md`
2. 更新同名 `AGENT.md`
3. 保证 `核心契约` 与 skill 完全一致
4. 运行同步脚本
5. 修复差异后再次验证

## 同步脚本

```bash
./scripts/sync-skill-agent.sh
./scripts/sync-skill-agent.sh tech-lead
```

脚本会检查：

- 是否存在同名 skill/agent
- 是否具备统一章节
- `核心契约` 是否一致

## 维护建议

- 普通业务 skill 不负责编排下游角色
- 只有协调器类 skill 可以描述下游协同
- 若 skill 行为边界变化，必须同步更新派生 agent
