# Skill vs Agent 映射规则

## 定义

- `SKILL.md`：完整角色说明与事实源
- `AGENT.md`：从白名单 skill 派生出的 subagent 逻辑源，平台运行时文件再由它生成

## 映射原则

| 维度 | SKILL.md | AGENT.md |
|------|----------|----------|
| 角色定位 | 完整 | 默认按最小变换保留对应章节 |
| 适用场景 | 完整 | 默认按最小变换保留对应章节 |
| 必须输入 | 完整 | 必须一致 |
| 可选输入 | 完整 | 必须一致 |
| 输出文件 | 完整 | 必须一致 |
| 执行规则 | 完整 | 必须一致 |
| 强制约束 | 存在同名 AGENT 时必须包含 | 必须一致 |
| 质量检查 | 完整 | 必须一致 |
| 下一步流程 | 完整 | 必须一致 |
| 其他强制约束段落 | 可包含 | skill 定义则必须一致 |
| 开场展示块 | 可包含 | 生成时移除 |

## 当前同步方式

当前直接以主章节作为同步锚点做精确比对，包括：

- `### 必须输入`
- `### 可选输入`
- `### 输出文件`
- `## 执行规则`
- `## 质量检查`
- `## 下一步流程`

同时继续对 `## 强制约束` 以及 skill 中声明的其他强制约束段落做精确对齐。

平台运行时的重生成和 clean-tree 校验见 [docs/scripts.md](/Users/yuqiyu/AiHistorys/team-collaboration-skills/docs/scripts.md)。

强制约束段落示例：

- `强制约束`
- `技术栈`
- `需求澄清机制`
- `设计确认机制`
- `设计原则`
- `计划输出重点`

## 何时需要同步

- 角色职责变化
- 输入输出变化
- 路径规则变化
- 质量门槛变化
- 下一步流转变化
- 强制约束段落变化

满足以上任一情况后，应重新运行 `./scripts/generate-agents-from-skills.sh`。
