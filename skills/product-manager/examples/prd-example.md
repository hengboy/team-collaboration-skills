# 需求背景

## 问题描述

当前登录流程存在以下问题：
1. 登录方式单一，仅支持账号密码
2. 忘记密码流程复杂，需要邮件申诉
3. 无验证码登录，用户忘记密码后无法快速登录

## 目标用户

- 新用户：未设置过密码
- 老用户：忘记密码
- 所有用户：希望快速登录

## 业务目标

- 提升登录转化率 15%
- 降低客服咨询量 30%
- 提升用户体验满意度

# 用户故事

| ID | 作为 | 我想要 | 以便于 | 优先级 |
|----|------|--------|--------|--------|
| US-001 | 用户 | 通过手机号 + 验证码登录 | 无需记忆密码快速登录 | P0 |
| US-002 | 用户 | 获取短信验证码 | 完成登录验证 | P0 |
| US-003 | 用户 | 60 秒后重新发送验证码 | 防止验证码未到达 | P1 |

| US-005 | 用户 | 登录成功后跳转到首页 | 继续使用系统 | P1 |

# 功能需求

## FR-001: 手机号登录

### 描述

用户通过手机号 + 6 位数字验证码完成登录。

### 验收条件

- [ ] 支持 11 位中国大陆手机号（13x/14x/15x/16x/17x/18x/19x）
- [ ] 验证码为 6 位数字
- [ ] 验证码有效期 5 分钟
- [ ] 同一手机号 60 秒内只能获取一次验证码
- [ ] 验证码错误时提示"验证码错误，请重新输入"
- [ ] 验证码过期时提示"验证码已过期，请重新获取"
- [ ] 登录成功后跳转回登录前页面或首页

## FR-002: 验证码获取

### 描述

用户点击"获取验证码"按钮后，系统发送短信验证码到用户手机。

### 验收条件

- [ ] 点击按钮前校验手机号格式
- [ ] 手机号格式错误时提示"请输入正确的手机号"
- [ ] 发送成功后按钮进入 60 秒倒计时
- [ ] 倒计时结束后显示"重新发送"
- [ ] 发送失败时显示错误原因

## FR-003: 登录状态保持

### 描述

用户登录成功后，可选择"7 天内自动登录"。

### 验收条件

- [ ] 默认不勾选自动登录
- [ ] 勾选后 7 天内无需重复登录
- [ ] 敏感操作（如修改密码）需重新验证
- [ ] 用户可手动退出登录

# 非功能需求

## 性能要求

- 登录接口 P95 延迟 < 500ms
- 验证码发送延迟 < 5s
- 支持 1000 QPS

## 安全要求

- 验证码尝试次数限制：5 次/小时
- 同一手机号发送频率限制：10 次/天
- 防刷机制：IP 维度限流
- 短信内容不含敏感信息
- 登录日志记录

## 监控要求

- 登录成功率监控
- 验证码发送成功率监控
- 异常登录行为告警
